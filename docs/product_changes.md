# Portal Experience: 产品功能与角色影响更新

## 🚀 核心功能更新 (Core Features)

1.  **反馈门户 (Feedback Portal)**
    *   新增 `/portal` 统一入口，聚合“提交反馈”、“路线图”、“变更日志”等功能模块。
    *   提供清晰的导航结构，替代原先零散的页面访问方式。

2.  **智能即时防重 (Inline Duplicate Detection)**
    *   用户在表单输入标题时，系统实时通过 AI/算法匹配现有反馈。
    *   在输入框下方即时显示“可能存在相似反馈”列表，支持点击查看。

3.  **智能链接预填充 (Smart URL Prefill)**
    *   支持通过 URL 参数自动填充表单字段。
    *   支持字段：`title` (标题), `description` (描述), `type` (类型: bug/feature/etc), `priority` (优先级)。

---

## 👥 角色影响详解 (Impact by Role)

### 1. 终端用户 (End Users)
*   **体验升级**: 拥有了一个集中的“反馈中心”首页，不再迷失。
*   **极速反馈**: 在输入标题时若看到相似问题，可直接点击查看并投票，**无需重复撰写**长篇大论，减少挫败感。
*   **操作简化**: 点击支持团队发送的特定链接时，表单类别、优先级等可能已被自动填好，只需补充核心信息。

### 2. 产品经理 & 管理员 (Product Managers & Admins)
*   **效率提升**: 系统在源头拦截了大量重复反馈，**Review 列表中的“重复项”将显著减少**，大大降低人工分拣和合并的压力。
*   **数据质量优化**: 通过推广带参数的链接（如 `?type=feature&title=二期需求`），可以引导用户提交更规范、分类更准确的数据。
*   **闭环体验**: 详情页新增“返回反馈中心”按钮，用户流失率降低，更容易留在门户内产生更多互动。

### 3. 客户支持团队 (Support Team)
*   **工单转化神器**: 当用户在聊天/邮件中报告问题时，支持人员可以直接发送预填充链接（例如：`/portal?type=bug&priority=high&title=登录失败`）。
*   **即用即走**: 降低用户填写门槛，提高工单收集成功率和信息的准确度。

### 4. 开发者 (Developers)
*   **API 能力**: 开放 `/api/feedback/similar` 端点。
*   **扩展性**: 可将“相似反馈搜索”功能集成到内部 CRM、CLI 工具或 Slack 机器人中，进一步打通工作流。
